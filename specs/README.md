# Specs (Spec Kit Lite)

This folder holds the spec-first artifacts for each skill/feature.

## Workflow

1) Create `specs/<feature>/SPEC.md`
- Write goal, scenarios, edge cases, interfaces, and **Definition of Done**.

2) Create `specs/<feature>/PLAN.md`
- Architecture + dependency choices + testing plan + decision log.

3) Create `specs/<feature>/TASKS.md`
- Small, verifiable tasks.

4) Implement
- Code and tests according to the plan.

5) Create `specs/<feature>/UAT.md`
- Provide a short checklist for the main agent/user.
